# AI Security Brief

> Intelligence on AI-Powered Threats & Privacy Defence

AI Security Brief is a Next.js publication covering AI-powered cyber threats, privacy tooling, and defensive guidance for technical readers and IT decision-makers.

The repository combines a markdown-backed editorial site, a server-side Beehiiv subscription flow, a tools directory with runtime-resolved affiliate links, and a GitHub Actions automation pipeline that writes to reviewable branches and draft PRs instead of pushing AI-generated output straight to `main`.

## Tech Stack

- **Framework:** Next.js 15 App Router
- **UI:** React 19 + Tailwind CSS 3
- **Language:** TypeScript
- **Content:** Markdown + frontmatter + generated `content-manifest.json`
- **Newsletter:** Beehiiv
- **Hosting:** Vercel
- **Automation:** GitHub Actions + GitHub Models

## Core Routes

## Content and Automation Model

- Articles live in `blog/*.md`.
- `content-manifest.json` is generated from article frontmatter and used by the site and verification scripts.
- Newsletter drafts are compiled into `drafts/` for review before publication.
- Editorial automation works on `codex/content-week-YYYY-WW`.
- Performance logging works on `codex/performance-week-YYYY-WW`.
- Automation opens or updates draft PRs so generated output is reviewed before merge.

### Workflow Schedule

All scheduled windows are enforced in `Australia/Sydney`.

| Day / time | Workflow | Output |
| --- | --- | --- |
| Sunday 20:00 | `performance-logger.yml` | Updates `logs/performance-log.md` on the performance branch |
| Monday 05:00 | `weekly-harvest.yml` | Writes `harvests/harvest-YYYY-MM-DD.md` |
| Monday 09:00 | `article-factory.yml` | Generates article drafts in `blog/` |
| Monday 13:00 | `newsletter-compiler.yml` | Generates a review-only newsletter draft in `drafts/` |
| Monday 15:00 | `seo-affiliate.yml` | Fills SEO gaps and applies affiliate placeholder work on the content branch |

All five workflows also support `workflow_dispatch` for manual runs. The harvest, article factory, and newsletter compiler flows also support a `force` regeneration flag.

## Prerequisites

- Node.js 20 or newer
- `pnpm` 10.23.0

If `pnpm` is not installed globally, you can use the pinned package manager via `npx pnpm@10.23.0`.

## Getting Started

```bash
git clone https://github.com/joshcabana/ai-security-brief.git
cd ai-security-brief
npx pnpm@10.23.0 install
cp .env.example .env.local
# Fill in the required environment variables
npx pnpm@10.23.0 dev
```

The app will be available at `http://localhost:3000`.

## Environment Variables

| Variable | Required | Purpose |
| --- | --- | --- |
| `BEEHIIV_API_KEY` | Yes for signup and performance logging | Beehiiv API authentication |
| `BEEHIIV_PUBLICATION_ID` | Yes for signup and performance logging | Beehiiv publication target |
| `NEXT_PUBLIC_SITE_URL` | Yes for `verify:ops` readiness | Canonical site URL |
| `NEXT_PUBLIC_SITE_NAME` | Yes for `verify:ops` readiness | Public site name |
| `AFFILIATE_NORDVPN` | Optional | Runtime link for NordVPN |
| `AFFILIATE_PUREVPN` | Optional | Runtime link for PureVPN |
| `AFFILIATE_PROTON` | Optional | Legacy Proton fallback |
| `AFFILIATE_PROTON_VPN` | Optional | Runtime link for Proton VPN |
| `AFFILIATE_PROTON_MAIL` | Optional | Runtime link for Proton Mail |
| `AFFILIATE_LINKS_PATH` | Optional | Override path for local affiliate link replacement mappings |
| `GITHUB_MODELS_MODEL` | Optional | Model override for local automation or GitHub Actions |

`verify:ops` also flags stale runtime variables such as `NEXT_PUBLIC_BEEHIIV_PUBLICATION_ID`, `SUPABASE_URL`, and `SUPABASE_ANON_KEY`.

## Verification and Commands

### Development

```bash
pnpm dev
pnpm build
pnpm start
```

### Content Verification

```bash
pnpm content:manifest
pnpm check:content
```

`pnpm check:content` verifies:

- article manifest sync
- newsletter draft issue numbers are unique
- newsletter draft links point to real article slugs
- affiliate live-status stays aligned across `ops/affiliate-status.md` and `ops/affiliate-intake.md`

### Release Verification

```bash
pnpm typecheck
pnpm test:unit
pnpm test:smoke
pnpm verify:release
```

`pnpm verify:release` runs the full local release gate:

1. `pnpm check:content`
2. `pnpm typecheck`
3. `pnpm test:unit`
4. `pnpm build`
5. `pnpm test:smoke`

`pnpm test:smoke` starts the production server, checks the key public routes, and exercises the subscribe API against a mock Beehiiv server.

### Ops Verification

```bash
pnpm verify:ops
pnpm verify:ops:contract
pnpm verify:live
pnpm ops:check-monday-pipeline -- --date YYYY-MM-DD
```

- `pnpm verify:ops` loads `.env.local` when present and checks required runtime variables plus banned stale variables.
- `pnpm verify:ops:contract` checks `.env.example` against the expected env contract.
- `pnpm verify:live` validates the deployed site.
- `pnpm ops:check-monday-pipeline` checks the expected open PRs and workflow runs for a Monday content cycle.

### Automation Commands

```bash
pnpm automation:weekly-harvest
pnpm automation:article-factory
pnpm automation:newsletter-compiler
pnpm automation:seo-affiliate
pnpm automation:performance-logger
```

## Affiliate Handling

- Article markdown can use `[Tool Name]([AFFILIATE:CODE])`.
- Runtime article rendering resolves `AFFILIATE_<CODE>` from environment variables and degrades unresolved links to plain text.
- The tools page resolves affiliate URLs by priority and falls back to clean vendor URLs when no tracked link is configured.
- Local placeholder replacement reads mappings in this order:
  1. `AFFILIATE_LINKS_PATH`
  2. `~/.ai-security-brief/affiliate-links.json`
  3. `ops/affiliate-links.json`

## Deployment

The canonical deployment workflow is `.github/workflows/deploy.yml`.

That workflow:

- runs verification on pushes and pull requests targeting `main`
- deploys to Vercel only when `VERCEL_TOKEN`, `VERCEL_ORG_ID`, and `VERCEL_PROJECT_ID` are present
- runs live verification against `https://aithreatbrief.com` on `main`
- uploads `verify-live.json` and `status.json` artifacts for post-deploy inspection
